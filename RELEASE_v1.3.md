# v1.3 - 性能优化与源文件清理增强

## 版本亮点
本版本聚焦“转换后清理”和“界面体验”，重点解决源目录残留、清理耗时长、状态反馈不清晰等问题，并增强打包脚本可用性与文档一致性。

## 新增与优化

### 1) 清理能力增强
- 新增更清晰的删除范围控制（可在设置中配置）
- 优化“删除源文件”逻辑，支持更完整地处理同级与上级目录清理场景
- 新增独立清理进度反馈，转换与清理阶段状态分离显示

### 2) 交互反馈优化
- 细化转换流程文案：可明确区分“正在转换 / 正在清理”等阶段
- 改善主界面交互细节，降低误解成本
- 更新主界面截图并同步文档引用

### 3) 运行与打包改进
- 增强 Windows 打包脚本可用性（便于双击/脚本方式打包）
- 同步 `release/` 目录构建产物与说明

### 4) 稳定性与测试
- UI 相关辅助逻辑补充测试覆盖
- 修复部分清理流程中的局部变量异常问题（如 `cleanup_done` 未绑定）

## 升级提示
- 建议在“设置”中确认删除范围与删除策略
- 首次升级后建议先用一个小样本目录验证“转换 + 清理”完整流程
- 如需保守使用，可先关闭自动删除，仅保留转换功能
